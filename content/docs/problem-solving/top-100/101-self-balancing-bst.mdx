## <TranslatedText en="Types of Self-Balancing BSTs" bn="স্ব-ভারসাম্যপূর্ণ BST এর ধরন" />

<TranslatedText
  en={
    <>
      <ul>
        <li><strong>AVL Trees:</strong> Strictly balanced, height difference between left and right subtrees is at most 1</li>
        <li><strong>Red-Black Trees:</strong> Less strictly balanced, uses coloring and rotation rules</li>
        <li><strong>Splay Trees:</strong> Self-adjusting, recently accessed elements near root</li>
        <li><strong>B-Trees:</strong> Multi-way search trees, commonly used in databases</li>
      </ul>
    </>
  }
  bn={
    <>
      <ul>
        <li><strong>AVL ট্রি:</strong> কঠোরভাবে ভারসাম্যপূর্ণ, বাম এবং ডান সাবট্রির উচ্চতার পার্থক্য সর্বোচ্চ 1</li>
        <li><strong>রেড-ব্ল্যাক ট্রি:</strong> কম কঠোরভাবে ভারসাম্যপূর্ণ, রঙ এবং ঘূর্ণন নিয়ম ব্যবহার করে</li>

        <li><strong>B-ট্রি:</strong> মাল্টি-ওয়ে সার্চ ট্রি, সাধারণত ডেটাবেসে ব্যবহৃত</li>

## <TranslatedText en="Why Self-Balancing?" bn="কেন স্ব-ভারসাম্যপূর্ণ?" />

<TranslatedText
  en={
    <>
      <ul>
        <li><strong>Worst Case Prevention:</strong> Regular BST can degrade to O(n) in worst case (skewed tree)</li>
        <li><strong>Guaranteed Performance:</strong> Self-balancing trees guarantee O(log n) operations</li>
        <li><strong>Real-World Applications:</strong> Used in databases, file systems, and memory management</li>
        <li><strong>Automatic Rebalancing:</strong> No manual intervention needed to maintain balance</li>
      </ul>
    </>
  }
  bn={
    <>
      <ul>
        <li><strong>সবচেয়ে খারাপ ক্ষেত্র প্রতিরোধ:</strong> সাধারণ BST সবচেয়ে খারাপ ক্ষেত্রে O(n) তে অবনতি হতে পারে (তির্যক ট্রি)</li>
        <li><strong>গ্যারান্টিড পারফরম্যান্স:</strong> স্ব-ভারসাম্যপূর্ণ ট্রি O(log n) অপারেশন গ্যারান্টি দেয়</li>
        <li><strong>বাস্তব-বিশ্ব অ্যাপ্লিকেশন:</strong> ডেটাবেস, ফাইল সিস্টেম এবং মেমরি ম্যানেজমেন্টে ব্যবহৃত</li>

      </ul>
