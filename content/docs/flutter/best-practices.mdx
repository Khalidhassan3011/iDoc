### Common Architecture Patterns

#### MVC (Model-View-Controller)

The MVC pattern separates your application into three main components:
- **Model**: Handles data and business logic
- **View**: Displays the user interface
- **Controller**: Manages the communication between Model and View

#### MVVM (Model-View-ViewModel)

MVVM is an evolution of MVC that provides better separation of concerns:
- **Model**: Data and business logic
- **View**: User interface
- **ViewModel**: Acts as a bridge between Model and View, handling UI logic

#### Redux

Redux provides predictable state management:
- Single source of truth (Store)
- State is read-only
- Changes are made through pure functions (Reducers)

## 2. Testing (Unit Testing, Integration Testing, Functionality Testing, etc)

### Why Testing Matters

As we all know, testing is an important point for developers (not just mobile, but web and all platforms).

> **Note:**
I am not talking here about Tester (QA). This testing is for developers - testing that is needed in our coding.

### Major Testing Categories

Testing is divided into some major categories:

1. **Unit Testing**
2. **Integration Testing**
3. **Functionality Testing**
4. **Performance Testing**
5. **User Interface Testing**

Also, there are some more testing types: smoke testing, alpha testing, system testing, stress testing, regression testing, etc.

### Let's Talk About Each Type

#### 1. Unit Testing

**In simple words**: Unit testing is to test each component or module of your software project.

It means they test small things like:
- Your if-else conditions
- Small operations like arithmetic and logical operations

```dart
// Example Unit Test in Flutter
test('Counter increments', () {
  final counter = Counter();
  counter.increment();
  expect(counter.value, 1);
});
```

#### 2. Integration Testing

This testing is focused on what output is produced after unit testing.

**In simple words**: They check the internal mechanisms of output.

```dart
// Example Integration Test
testWidgets('Counter increments smoke test', (WidgetTester tester) async {
  await tester.pumpWidget(MyApp());

  expect(find.text('0'), findsOneWidget);
  expect(find.text('1'), findsNothing);

  await tester.tap(find.byIcon(Icons.add));
  await tester.pump();

  expect(find.text('0'), findsNothing);
  expect(find.text('1'), findsOneWidget);
});
```

#### 3. Functionality Testing

This type of testing checks if the system performs or behaves as per requirements or not.

**Key aspects**:
- Does the feature work as expected?
- Does it meet business requirements?
- Are all user flows working correctly?

#### 4. Performance Testing

This type of testing measures your project's:
- Performance
- Speed
- Network stability
- All factors required for better performance

**What to measure**:
- App startup time
- Screen render time
- Memory usage
- Network response time
- Battery consumption

#### 5. User Interface Testing

This testing checks how your project performs in the graphical user interface, or we can say it's responsive design.

**In simple words**: How it performs on live devices, because there are various sizes and screen densities. We need our project to perform well everywhere.

## 4. Always Train Yourself for DSA (Data Structures and Algorithms)

### Why DSA is Important

We all know this is a very interesting topic for every developer.

During interviews, it is mostly highlighted.

> **Note:**
No one is perfect at this, and everyone wants to be a perfectionist.

### But Why?

The reason is: **Every great application depends on how it works perfectly with user requirements.**

For these requirements, we (developers) write great logic that fulfills the requirements of the application with proper code.

### Real-World Example

Because of every sort of collection data, we need to filter based on requirements.

**For example**:

If we have 1,000 user data entries, we need to separate this data by:
- Age criteria
- Gender criteria
- Location criteria
- Any other filters

### What We Do

We write logic, and this logic analyzes how we will filter in terms of:
- **Time Complexity**: How fast the algorithm runs
- **Space Complexity**: How much memory it uses

### Understanding DSA

> **Note:**
**Data Structure** is like a named location (where you store your data).

**Algorithm** is a collection of steps to solve a particular problem.

Learning data structures and algorithms allows us to write:
- âœ… Efficient code
- âœ… Well-structured code
- âœ… Optimized programming

### Common Data Structures

1. **Arrays/Lists**: Ordered collection of elements
2. **Maps/HashMaps**: Key-value pairs
3. **Sets**: Unique elements collection
4. **Stacks**: Last-In-First-Out (LIFO)
5. **Queues**: First-In-First-Out (FIFO)
6. **Trees**: Hierarchical data structure
7. **Graphs**: Network of nodes and edges

### Algorithm Categories

1. **Sorting Algorithms**: QuickSort, MergeSort, BubbleSort
2. **Search Algorithms**: Binary Search, Linear Search
3. **Dynamic Programming**: Optimization problems
4. **Greedy Algorithms**: Making locally optimal choices
5. **Recursion**: Function calling itself
6. **Graph Algorithms**: BFS, DFS, Dijkstra's

### Example in Flutter

```dart
// Example: Filtering a list of users by age
class User {
  final String name;
  final int age;
  final String gender;

  User(this.name, this.age, this.gender);
}

List<User> filterUsersByAge(List<User> users, int minAge, int maxAge) {
  // Time Complexity: O(n)
  // Space Complexity: O(n)
  return users.where((user) =>
    user.age >= minAge && user.age <= maxAge
  ).toList();
}

List<User> sortUsersByAge(List<User> users) {
  // Time Complexity: O(n log n)
  // Space Complexity: O(n)
  users.sort((a, b) => a.age.compareTo(b.age));
  return users;
}
```
